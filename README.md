# ZeroRT

These patchset provides suitable patches to mmake OPIZ2 compatible with mainline kernel, up to 6.12.x. Official SUNXI images are based on pre 5.x kernels with proprietary drivers and binary blobs which are not compatible with modern kernels or recent versions of musl/glibc. These patchset provides GPU, WiFi, bluetooth, HDMI and additional CPU subsystem capabilities.
Most CPU subsystem patches, crypto and HDMI modules for example, are derived from the sun50i family which shares the same allwinner architecture. For this board, a chinese made WiFi+bt combo chip is provided, and its drivers were extracted from deprecated SUNXI build scripts and patched to be compatible with modern kernels and preemtive realtime scheduling.
SUNXI images use proprietary Arm blobs for these Mali chips. recently, Mesa has added support for this family via open-source Panfrost drivers. The upstream SUNXI U-Boot builds used to set the GPU_RESET register on power rail on boot. Upstream U-boot now supports OPIZ2, but doesn't use this adhoc method. It turns out that without setting this register, if the Panfrost driver tries to wake the GPU up, or put it to low power mode (not tested), the entire system freezes. So, the upstream SUNXI driver for sun50i power management module was patched to set the GPU_RESET register for Panfrost to work properly.
